3-Hour Session Structure

Part 1 — Business Walkthrough (~90 min)
Harbor cruise Dock through Stop 8. Business-level narrative for all attendees. Product, strategy, design as primary audience.
Part 2 — Engineering Deep-Dive (~60 min)
Open the hood: Data Cloud data model, SDLC/sandbox lifecycle, DevOps/Change Sets, data ingestion patterns. Engineers and architects stay.
Part 3 — Office Hours / Q&A (~30 min)
Open floor. Deeper dives on any stop. Whiteboard session if needed. Leave-behind resources distributed.
Leave-behind: The MC lifecycle/sandbox guide will be shared as a URL — participants can reference and share it after the session.

Storyline

CarMax marketers launch lifecycle campaigns in MCA. They draft with Campaign Agent, build audiences using Einstein Segmentation, enrich and prioritize with Calculated Insights, engagement data, and waterfall logic — which directly solves the "one person getting every email" problem that today requires complex DE tracking and journey exit criteria. They deliver web-engagement-triggered dynamic email content (Figma components mapped to MCA email builder), orchestrate cross-channel with SMS in MCA Flow, analyze outcomes with reporting, and close with a dynamic feedback loop — including hearting a vehicle directly from within an email.

Marketer Speed AI-Assisted Build Less Dev-Dependent Waterfall Prioritization Dynamic Email Figma → MCA Components SMS In MCA Flow Customer 360 Reporting Loop In-Email Heart Action

Harbor Cruise Run of Show — Part 1 Business Walkthrough

Dock: Business Framing
5 min
  • Welcome and introductions
  • Set business objective: increase test drive bookings from vehicle browsers
  • Preview the 9-stop harbor cruise format
  • Introduce key personas: Anthony Parker (active shopper with EV interest) and Aisha Thompson (new subscriber)
  • Frame the session: new builds happen on MCA — this isn't about replacing MCE overnight
Acknowledge up front: today's MCE complexity — DE tracking, exit criteria, dev-heavy journey logic — is the problem this demo solves. Set the "before" picture before showing the "after."
Stop 1: Campaign Agent & Brief
12 min
  • Draft campaign from business brief using AI agent
  • Walk through generated brief: audience, channels, messaging, metrics
  • Show brand voice and tone settings
  • Connect brief to Campaign record — agent also generates segment description
  • Live prompt: "Create a campaign targeting customers who hearted vehicles but haven't purchased. Use email as primary channel. Warm CarMax tone. Dynamic vehicle cards with favorites, pricing, and test drive CTA."
  • Key point: "The marketer didn't write SQL or build a segment — they described the business intent. The agent generates not just the brief but a description of the segment to match."
  • Audience note: For strategists and PMs — this is the "speed" story. For designers — this is where content requirements feed the email build.
  • Fallback: Show pre-created brief if Campaign Agent is slow
  • Transition: "Now that we have our brief — and a segment description — let's build the audience it describes."
Stop 2: Segmentation + CIs + Waterfall
18 min
  • Part A (6 min): Einstein natural-language segmentation — show EV Consideration audience discovery
  • Part B (5 min): Calculated Insight enrichment — CLV, Propensity, Preference, Velocity scores on Anthony's profile
  • Part C (7 min): Waterfall prioritization — Anthony in 3 segments resolves to Priority 1 only
  • "Aha moment": Anthony Parker qualifies for ALL 3 tiers — Hearted Vehicle (P1), Test Drive (P2), Memorial Day (P3) — but waterfall assigns P1 only
  • Connect to the "before" pain point: "Today CarMax tracks this with data extensions and journey exit criteria. One person can accidentally receive every campaign simultaneously. The waterfall is a platform-level solution — the logic is defined once, enforced automatically, no DE maintenance required."
  • Christine's exact pain point: "We consider what other emails they already got and what actions they've taken before deciding what to send or not send." — The waterfall makes this automatic rather than manually maintained.
  • Key point: "Without the waterfall, Anthony would receive 3 separate campaigns. With it, he gets the single most relevant message — and the platform enforces it, not a developer."
  • Transition: "Anthony is now in the Hearted Vehicle segment. Let's see what email he'll receive."
Stop 3: Dynamic Email + Data Graph
14 min ⭐
  • Part A (5 min): Email structure — modular design with component + HTML blocks
  • Part B (4 min): Data Graph connection — CarMax Customer 360 powers the repeater
  • Part C (5 min): Persona previews — Anthony (hearted vehicles + EV hero banner) vs Aisha (fallback + onboarding blocks)
  • Figma → MCA component callout (for designers): "Today your designers work in Figma and map every component to a Content Builder component in SFMC — devs use Figma's developer mode to find the component name, then find it in SFMC and drag-and-drop. In MCA, that same component-based model carries forward. The translation from Figma component to MCA email builder component is a direct analogy."
  • Anthony preview: EV hero banner (Data Graph detects EV interest signals), 3 dynamic vehicle cards from hearted vehicles, pre-qual block hidden, "Schedule a Test Drive" CTA
  • Aisha preview: Default hero banner (no EV interest, no hearted vehicles), "Browse Our Top Picks" fallback, "Create an Account" + "Get Pre-Qualified" blocks visible, "Start Browsing" CTA
  • Key point: "Same template, completely different experience — true 1:1 personalization at scale. The hero banner, the vehicles, every block — all driven by what the Data Graph knows about that individual."
  • Transition: "Now let's see how this email gets delivered — and what happens when Anthony engages or doesn't."
Stop 4: MCA Flow + SMS
14 min
  • Part A (7 min): Flow canvas walkthrough — segment entry → wait → suppression → email → wait for click → branch to SMS or reminder
  • Part B (5 min): Live SMS authoring with Data Cloud personalization tokens
  • Part C (2 min): Quick overview of supporting flows (Welcome, Price Drop, Instant Offer, Saved Search)
  • Marketer role callout: "Flow is the new Journey Builder — but when a marketer logs in with the marketer role, they only see marketer-relevant actions: send an SMS, did someone click an email. They don't see the full complexity of everything Flow can do on Core. It's the same platform, filtered to what makes sense."
  • For engineers — Eric's point: "Because Flow is part of Salesforce Core, it's not siloed to marketing. If you need to route a contact to someone in a store for a human follow-up, that's the same data, the same flow. No separate system required."
  • The "magic" moment — address Christine's pain point directly: "Today, managing suppression and send frequency in MCE requires data extensions, exit criteria defined in every journey, and dev involvement to keep them in sync. In MCA, the suppression logic lives in the flow, the waterfall handles priority, and the marketer can configure these rules themselves — it's less dev-dependent by design."
  • Flow logic: Segment Entry → Wait 2hr → Decision (suppress purchasers) → Send Email → Wait for Click (3 day) → Branch (opened → SMS, not opened → reminder)
  • SMS content: "Hi {{FirstName}}! Still thinking about the {{Vehicle_Year}} {{Vehicle_Make}} you test drove at CarMax {{Store}}? Your 24-hour test drive is still available. Book again: {{ShortLink}} Reply STOP to opt out."
  • Key point: "SMS and email work together in one flow — no separate journey for each channel. And suppression is enforced at the platform level, not maintained in a spreadsheet."
  • Transition: "Let's look at how Data Cloud ties it all together."
Stop 5: Data Cloud Customer 360
8 min
  • Anthony Parker's unified profile — CRM + web + engagement + AI scores (incl. EV interest signals)
  • Data Graph visualization: Individual → Vehicle, Test Drive, Web Engagement
  • Connect back: "Everything the email personalized, the flow decided, the segment qualified — all from this profile"
  • Note: browsing history (viewed car page) and pre-qual/instant offer actions are either in Salesforce or being moved there — this profile will only get richer
Stop 6: Reporting & Optimization
7 min
  • Campaign-level metrics: sends, opens, clicks
  • Flow branch analytics: suppressed, engaged, SMS vs email
  • Identify optimization opportunity from branch performance
Stop 7: Dynamic Feedback Loop
6 min
  • Present feedback loop concept: email interaction → Data Cloud update → next email smarter
  • Hearting a vehicle directly from within the email — customer clicks heart on a vehicle card in the email; that interaction writes back to Data Cloud and influences the next send
  • Connect to existing architecture — Data Graph + dynamic loop + click tracking
  • "Each interaction makes the next one more relevant — including actions taken inside the email itself"
This stop is conceptual — no live demonstration of the writeback loop. But hearting-in-email was specifically called out by the CarMax team as a high-value example to feature prominently.
Stop 8: Wrap + Executive Recap
4 min
  • Restate: speed, intelligence, precision, scale, control
  • Tie back to the "before": complex DE tracking, dev-dependent suppression, separate journeys per channel → all simplified
  • "MCA isn't just a new tool — it's the platform removing the overhead that today keeps marketing from moving fast."
  • Transition to Part 2 (engineering deep-dive) or Q&A
Buffer: Q&A / Transition
2 min
  • Field questions from Part 1 audience before engineering deep-dive begins
  • Note for the room which attendees are staying for Part 2

Readiness Checklist

Use these checkboxes before the live session starts.

Risks & Mitigations

  • Campaign Agent slow: Show pre-created brief as fallback
  • Segment membership empty: Show segment rules and explain logic
  • Email preview broken: Show email in builder mode (component view)
  • Data Graph not built: Show definition and explain architecture
  • Flow not activating: Walk through canvas in Draft mode
  • SMS channel not configured: Focus on flow walkthrough, skip live SMS
  • Query latency: Pre-open all Data Cloud tabs
  • Sparse reporting data: Use seeded/historical examples
  • Session timeout: Log in fresh 15 min before
  • Audience asks about future features (262/summer): Acknowledge and clearly flag as upcoming — do not imply it's available today
  • Scope drift into migration strategy: Park it for a separate conversation; keep Part 1 focused on "new builds on MCA"
  • Engineering audience derails Part 1 with deep technical questions: Acknowledge, note for Part 2 agenda, keep moving
