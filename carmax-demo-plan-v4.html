Storyline

CarMax marketers launch lifecycle campaigns in MCA. They draft with Campaign Agent, build audiences using Einstein Segmentation, enrich and prioritize with Calculated Insights, engagement data, and waterfall logic, then deliver web-engagement-triggered dynamic email content, orchestrate cross-channel with SMS in MCA Flow, analyze outcomes with reporting, and close with a dynamic feedback loop vision.

Marketer Speed AI-Assisted Build Waterfall Prioritization Dynamic Email SMS In MCA Flow Customer 360 Reporting Loop
Keep this positioning line consistent: Data Cloud supports the decisioning layer; MCA is the execution center, with web engagement already stream-fed for activation.

Harbor Cruise Run of Show (90 min)

Dock: Business Framing
5 min
  • Welcome and introductions
  • Set business objective: increase test drive bookings from vehicle browsers
  • Preview the 9-stop harbor cruise format
  • Introduce key personas: Jane Dawson (power shopper) and Aisha Thompson (new subscriber)
Stop 1: Campaign Agent & Brief
12 min
  • Draft campaign from business brief using AI agent
  • Walk through generated brief: audience, channels, messaging, metrics
  • Show brand voice and tone settings
  • Connect brief to Campaign record
  • Live prompt: "Create a campaign targeting customers who hearted vehicles but haven't purchased. Use email as primary channel. Warm CarMax tone. Dynamic vehicle cards with favorites, pricing, and test drive CTA."
  • Key point: "The marketer didn't write SQL or build a segment — they described the business intent"
  • Fallback: Show pre-created brief if Campaign Agent is slow
  • Transition: "Now that we have our brief, let's build the audience it describes."
Stop 2: Segmentation + CIs + Waterfall
18 min
  • Part A (6 min): Einstein natural-language segmentation — show EV Consideration audience discovery
  • Part B (5 min): Calculated Insight enrichment — CLV, Propensity, Preference, Velocity scores on Jane's profile
  • Part C (7 min): Waterfall prioritization — Jane in 3 segments resolves to Priority 1 only
  • "Aha moment": Jane Dawson qualifies for ALL 3 tiers — Hearted Vehicle (P1), Test Drive (P2), Memorial Day (P3) — but waterfall assigns P1 only
  • Key point: "Without the waterfall, Jane would receive 3 separate campaigns. With it, she gets the single most relevant message."
  • Transition: "Jane is now in the Hearted Vehicle segment. Let's see what email she'll receive."
Stop 3: Dynamic Email + Data Graph
14 min ⭐
  • Part A (5 min): Email structure — modular design with component + HTML blocks
  • Part B (4 min): Data Graph connection — CarMax Customer 360 powers the repeater
  • Part C (5 min): Persona previews — Jane (3 vehicle cards) vs Aisha (fallback + onboarding blocks)
  • Jane preview: 3 dynamic vehicle cards (Ford Expedition XLT, Toyota 4Runner SR5, Hyundai Santa Fe SEL), pre-qual block hidden, "Schedule a Test Drive" CTA
  • Aisha preview: "Browse Our Top Picks" fallback, "Create an Account" + "Get Pre-Qualified" blocks visible, "Start Browsing" CTA
  • Key point: "Same template, completely different experience — true 1:1 personalization at scale"
  • Transition: "Now let's see how this email gets delivered — and what happens when Jane engages or doesn't."
Stop 4: MCA Flow + SMS
14 min
  • Part A (7 min): Flow canvas walkthrough — segment entry → wait → suppression → email → wait for click → branch to SMS or reminder
  • Part B (5 min): Live SMS authoring with Data Cloud personalization tokens
  • Part C (2 min): Quick overview of supporting flows (Welcome, Price Drop, Instant Offer, Saved Search)
  • Flow logic: Segment Entry → Wait 2hr → Decision (suppress purchasers) → Send Email → Wait for Click (3 day) → Branch (opened → SMS, not opened → reminder)
  • SMS content: "Hi {{FirstName}}! Still thinking about the {{Vehicle_Year}} {{Vehicle_Make}} you test drove at CarMax {{Store}}? Your 24-hour test drive is still available. Book again: {{ShortLink}} Reply STOP to opt out."
  • Key point: "SMS and email work together in one flow — no separate journey for each channel"
  • Transition: "Let's look at how Data Cloud ties it all together."
Stop 5: Data Cloud Customer 360
8 min
  • Jane Dawson's unified profile — CRM + web + engagement + AI scores
  • Data Graph visualization: Individual → Vehicle, Test Drive, Web Engagement
  • Connect back: "Everything the email personalized, the flow decided, the segment qualified — all from this profile"
Stop 6: Reporting & Optimization
7 min
  • Campaign-level metrics: sends, opens, clicks
  • Flow branch analytics: suppressed, engaged, SMS vs email
  • Identify optimization opportunity from branch performance
Stop 7: Dynamic Feedback Loop
6 min
  • Present feedback loop concept: email interaction → Data Cloud update → next email smarter
  • Connect to existing architecture — Data Graph + dynamic loop + click tracking
  • "Each interaction makes the next one more relevant"
This stop is conceptual — no live demonstration of the writeback loop.
Stop 8: Wrap + Executive Recap
4 min
  • Restate: speed, intelligence, precision, scale, control
  • "MCA isn't just a migration — it's a step change in what your marketing team can do."
Buffer: Q&A
2 min
  • Field audience questions
  • Offer deeper dives on any stop

Readiness Checklist

Use these checkboxes before the live session starts.

Risks & Mitigations

  • Campaign Agent slow: Show pre-created brief as fallback
  • Segment membership empty: Show segment rules and explain logic
  • Email preview broken: Show email in builder mode (component view)
  • Data Graph not built: Show definition and explain architecture
  • Flow not activating: Walk through canvas in Draft mode
  • SMS channel not configured: Focus on flow walkthrough, skip live SMS
  • Query latency: Pre-open all Data Cloud tabs
  • Sparse reporting data: Use seeded/historical examples
  • Session timeout: Log in fresh 15 min before
