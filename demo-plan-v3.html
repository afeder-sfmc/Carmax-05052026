Phase 3 ⭐

Upper-Funnel Nurturing — Audience & Segment Creation

⏱ 20 minutes · CENTERPIECE 0:30 – 0:50
Pain: Strategists → Snowflake → data team → days Goal: Done in Salesforce, in minutes No SQL · No engineering ticket · No Snowflake login

📋 The Use Case — Sean's Own Words

"The big one we're working on now, which is a huge value driver, is upper-funnel nurturing. It's people that have done X, Y, and Z but haven't progressed. We could even do [things like] those things and they've clicked on the same car five days in a row — we would have that activity in Snowflake. Showing that capability would be a huge win."
— Sean Goetz, April 10, 2026

Segment logic:

✓Downloaded the CarMax mobile app
✓Favorited a car
✓Clicked on the same car 5+ days in a row
✗Has not submitted a financing application

Why it matters:

▸ Repeated engagement with the same vehicle = high-intent signal

▸ No financing submission = friction point, not disinterest

▸ Currently requires Snowflake window function + data team handoff. In MCA: built by a strategist in under 10 minutes.

1

⭐ Open Segment Builder — Set the Persona (2 min)

Navigate to Data Cloud → Segments → New Segment. Emphasize who is doing this: a marketing strategist, not a data engineer.

Opening script: "I'm going to play the role of one of your strategists — someone who today has to go ask the data team to run queries in Snowflake. Watch how long this takes."

2

⭐ Build Filters 1–2 — App Download + Favorited (3 min)

Add filter: Mobile App Download Event exists. Add filter: Favorites Event exists (count ≥ 1). Pause and note the count updating in real time as each filter is applied.

Talking point: "Your strategist can see how many people match as they add filters. No waiting for a query to run."

3

⭐⭐ Build Filter 3 — Same Car, 5 Days in a Row (5 min)

This is the high-intent signal Sean specifically described. The most complex filter — show it carefully.

⭐ The Money Moment — Linger Here

Car Detail Page View — same Vehicle ID — Count ≥ 5 — within last 14 days. In Snowflake: a window function. In MCA Segment Builder: a filter configuration.

Script: "This is the signal Sean described — someone who has looked at the same car five days in a row. In Snowflake, this is a window function. In here, it's a dropdown. Same result. Different world."

4

⭐ Build Filter 4 — Has NOT Submitted Financing (2 min)

Add exclusion: Financing Application Submitted does NOT exist. The "not yet converted" gate that makes this upper-funnel vs. active buyer.

Talking point: "We're not emailing people who already applied — that's noise. We want the people who are clearly interested but haven't taken the step yet."

🙋 Anticipated Questions — Have Answers Ready

Q:

"Our data isn't all in Salesforce — a lot is still in Snowflake."

A: "Data Cloud ingests your Snowflake activity data via a data stream. Once it's mapped, your strategists access it through this same interface. The data stays in Snowflake as the source of truth. Your strategists just never need to touch it."

Q:

"How does this handle the 'same car' logic — that feels like it needs SQL?"

A: "For standard aggregations, Segment Builder handles it via the data model — Vehicle ID is a field on the Page View event, so 'same vehicle' is just filtering on that field. For more complex aggregations, Calculated Insights let a data person define the metric once in SQL, and then it shows up as a simple filter for the strategist."

Q:

"How often does the segment refresh?"

A: "Segment membership updates continuously as data changes. When a financing application is submitted, that person exits the segment. No manual re-pull required."

1

⭐ Frame the Problem (1 min)

Script: "Every email you've sent in ExactTarget is a dead end. The customer opens it, clicks a link, maybe visits the site — but that click doesn't update their profile, it doesn't update their favorite vehicles list, and it doesn't change what the next email looks like. MCA changes that."

2

⭐⭐ Show the Heart Link in the Email (2 min)

Open the email built in Module 2. Show the vehicle card — each car has a tracked heart icon link. The URL carries two parameters: subscriber key and vehicle ID. This is the standard MCA tracked link + a custom parameter.

https://[dc-ingestion-endpoint]/v1/ingest/heart?
  subscriberKey={{root.$dataGraph.ContactKey__c}}
  &vehicleId={{root.$dataGraph.VehicleId__c}}
  &action=favorite

Talking point: "That link hits a Data Cloud Streaming API endpoint. It's a real-time ingestion call. No CloudPage, no custom redirect, no identity stitching workaround — the subscriber key is already in the email context."

3

⭐⭐⭐ Click the Heart — Watch the Profile Update (4 min)

Click the heart link in the test email. Switch to the Data Cloud profile for that test customer. Show the Unified Profile — the favorited vehicle now appears in their Vehicle Favorites related records.

⭐ The "Aha" Moment — Let It Land

The customer clicked a heart inside an email. Their profile updated in real time. No website visit. No separate API call from the CarMax app. The email itself was a data input. The next email this person receives can now reference the vehicle they hearted — because it's in their profile.

Script: "That click — from inside an email — just wrote to their Data Cloud profile. Their next segment evaluation, their next triggered flow, their next email: they'll all reflect that favorite. The email is no longer a dead end. It's a data source."
